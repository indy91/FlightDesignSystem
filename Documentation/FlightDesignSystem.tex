\section{Introduction}

The Flight Design System is a ground-up rendezvous mission planning tool, specifically for the Orbiter addons NASSP and SSV. Launch time and targeting parameters can be calculated and verified as well as the rendezvous profile.\\

\newpage
\section{Config}
For a new project, the first step should always be to enter the data on the config tab and save. At this point a folder for the project is created under Projects. The year, month and day should be for the launch day. The world selection affects Earth rotation and coordinate system conversions. The "real" world uses parameters for the actual Earth, including nutation. All displayed times in the FDS are then in Coordinated Universal Time (UTC).  Selecting NASSP or SSV will make use of the Orbiter conversion functions between coordinate systems.  Orbiter uses Barycentric Dynamical Time (TDB) as its time system and all displayed times in the FDS will be in TDB with NASSP or SSV selected. NASSP does not use the standard Earth configuration file in Orbiter, so the World selection accounts for certain differences in Earth radius and rotations. \\
The last parameter on this tab is the difference between these two time measuring systems, UTC and TDB. This should be entered for all "worlds", but it is especially important for the Orbiter compatible modes (NASSP or SSV) as TLEs are in UTC and they need to be converted to the Orbiter time measuring system. Also with the "real" world selection this number is used for certain precession/nutation/rotation calculations. The value can be manually entered, but for convenience a button has been added to automatically generate the number based on the entered date. The time is is fixed offset (32.184 seconds) plus the number of leap seconds since 1972.\\

\section{State Vector}
State vectors can be added in a few formats and coordinate systems. The typical state vector for a rendezvous target will be in the TLE format, while internally stored state vectors will be in the True Equator-Greenwich (TEG) coordinate system. Also supported are J2000 and M50. While TLE data already contain information about the drag of the vehicle, the FDS uses numerical integration for trajectory propagation, so each state vector additionally needs weight, vehicle area and a K-factor for drag calculations. The K-factor is a multiplier that as used with a hardcoded drag coefficient (CD) of 2.0. In other places in the FDS the CD is entered directly.\\
The format of the SV text box is as follows:\\

\begin{center}
\begin{tabular}{|l|l|l|l|l| }
\hline
-&TEG&J2000&M50&TLE\\
\hline
Line 1&Pos XYZ in m/s&Pos XYZ in m/s (lh)&Pos XYZ in m/s&TLE Line 1\\
\hline
Line 2&Vely XYZ in m/s&Vel XYZ in m/s (lh)&Vel XYZ in m/s&TLE Line 2 \\
\hline
Line 3& GMT in seconds&MJD in days&MJD in days&-\\
\hline
\end{tabular}
\end{center}

\section{Walkthrough: Shuttle rendezvous mission}
Enter project name, select SSV as world, enter Year, Month and Day of launch. Save the project. Find a TLE for the launch day, best just before liftoff time. Go to the State Vector tab, click New and enter the full TLE under SV. Change the Coordinates selection to TLE, enter weight, area and K-Factor (typically 1.0) and save. Click check to see how close to midnight of launch day the TLE time tag is.\\
